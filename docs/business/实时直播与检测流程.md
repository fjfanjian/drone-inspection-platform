# 实时直播与检测流程

## 1. 业务目标

在无人机执行任务或人工控制过程中，平台需要提供实时画面、实时 AI 检测结果和异常告警能力，并将结果沉淀为可追溯业务记录。

## 2. 涉及角色

- 巡检人员
- 平台管理员
- 指挥或监管人员
- 检测服务运维人员

## 3. 前置条件

- 设备已在 DISys 中完成接入并可用
- 前端已获得有效登录态和工作空间上下文
- 视频流可通过 SRS 获取
- Detection Server 可访问模型、SRS、MinIO 和 DISys

## 4. 主流程

1. 用户在前端进入飞控或检测展示页面，选择设备或直播流。
2. 前端通过 DISys 获取设备、任务、工作空间和流地址相关信息。
3. 无人机或机场将视频流推送至 SRS。
4. 前端使用 HTTP-FLV 或其他前端流方式展示原始直播流。
5. 用户在前端发起实时检测，前端调用 Detection Server 的检测启动接口。
6. Detection Server 拉取 SRS 中的视频流，加载指定模型或检测组，创建检测会话。
7. Detection Server 将检测会话信息回调或登记到 DISys。
8. Detection Server 持续进行逐帧推理，并通过 WebSocket 向前端推送检测结果与状态更新。
9. Detection Server 根据策略保存截图、录像或结构化结果，并上传至 MinIO。
10. Detection Server 将检测结果摘要、报告任务或媒体元数据回调给 DISys。
11. 前端实时展示检测框、检测统计、会话状态和告警信息。

## 5. 异常流程

- SRS 拉流失败：检测会话进入异常状态，前端提示无法启动或已中断
- 模型加载失败：Detection Server 直接拒绝启动请求
- WebSocket 中断：前端失去实时结果流，必要时需要轮询兜底
- MinIO 上传失败：实时检测可继续，但媒体或报告可能不完整
- DISys 回调失败：检测执行继续，但业务记录可能延迟或缺失

## 6. 涉及页面与接口

### 主要页面

- `DroneCloudSystem-web/src/views/DetectionDisplay.vue`
- `DroneCloudSystem-web/src/views/FlightControl.vue`

### 主要接口

- 前端 -> DISys：设备、任务、媒体、会话展示相关接口
- 前端 -> Detection Server：启动检测、停止检测、查询状态、查询历史结果
- Detection Server -> DISys：会话注册、告警上报、媒体登记、报告回调

## 7. 关键业务约束

- 前端与 DISys 使用 `x-auth-token`
- Detection Server 对 DISys 也走 JWT 登录和 `x-auth-token` 头
- 前端调用业务接口前通常需要先确认 `workspaceId`
- Detection Server 会话状态当前以内存为主，服务重启后会丢失运行态
