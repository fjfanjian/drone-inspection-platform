# 业务总览

## 1. 平台定位

无人机智慧巡检平台是一个基于 DJI Cloud API 的行业级远程巡检平台，围绕“设备接入、任务执行、实时视频、AI 检测、告警处置、报告归档”形成闭环。

平台不是单一系统，而是由三个独立 Git 仓库共同组成的业务整体：

- `DISys`：业务主后端
- `DroneCloudSystem-web`：Web 控制台
- `DroneCloudSystem_detection-server`：AI 检测服务

## 2. 主要业务角色

- 平台管理员：管理用户、设备、工作空间、权限、系统配置
- 巡检人员：执行巡检任务、查看检测结果、处理告警
- 指挥或监管人员：关注实时画面、告警、任务完成情况
- 算法或运维人员：维护检测模型、检测服务、流媒体与联调环境

## 3. 核心业务对象

- 工作空间：多租户或多项目隔离边界
- 租户：平台服务的客户组织，当前与工作空间一一对应
- 项目：租户内部的业务组织单元
- 班组：项目下的巡检执行组织
- 设备：机场、无人机、负载设备等受控实体
- 航线：预设飞行路径与巡检任务配置
- 巡检任务：一次具体执行的飞行或巡检安排
- 检测会话：一次 AI 实时或离线检测运行实例
- 告警：检测或业务规则触发的异常事件
- 媒体文件：直播流截图、录像、检测图、报告等产物
- 报告：巡检结果或检测结果的结构化输出

## 4. 业务闭环

典型闭环如下：

1. 管理员完成设备接入、工作空间配置和权限初始化。
2. 业务人员创建或选择航线，发起巡检任务。
3. 无人机执行任务并产生视频流、遥测、事件消息。
4. 前端展示实时画面，检测服务对视频流进行实时推理。
5. 检测结果通过 WebSocket 推送到前端，并按规则触发告警。
6. 检测服务将截图、视频和报告上传到对象存储，并回调后端登记业务数据。
7. 巡检人员或管理人员查看结果、处理告警、归档任务。

## 5. 子系统职责边界

### DISys

- 用户认证与权限控制
- 工作空间、设备、航线、任务等业务主数据
- 租户、项目、班组和 RBAC 角色权限
- 告警、媒体、报告等业务记录持久化
- 与 DJI 设备链路和基础设施的主业务编排

### DroneCloudSystem-web

- 页面编排与用户交互
- 任务配置、飞控展示、检测面板、告警面板
- 与 DISys、Detection Server、MQTT、WebSocket 的多通道交互

### DroneCloudSystem_detection-server

- 模型管理与推理执行
- 实时检测、离线检测、多模型组合检测
- 截图、录像、检测结果推送、报告生成
- 将检测业务结果回调至 DISys

## 6. 平台级文档的维护边界

以下内容应优先维护在平台级文档中：

- 跨两个以上仓库的业务流程
- 跨服务状态流转
- 平台统一术语定义
- 联调约束、端口矩阵、启动顺序
- 版本清单与跨仓库变更记录
