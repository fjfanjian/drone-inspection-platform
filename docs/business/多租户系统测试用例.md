# 多租户系统测试用例

## 1. 测试目标

本文基于《多租户系统业务文档》编写，用于验证无人机智慧巡检平台在多租户场景下的组织、权限、设备、任务、媒体和实时消息隔离能力。

核心目标：

1. 同租户用户只能访问本租户工作空间内的数据。
2. 跨租户猜测 `workspace_id`、项目 ID、班组 ID、用户 ID、设备 SN、任务 ID、媒体 ID 时不能获取真实数据。
3. 机场、无人机、负载、OSD、在线状态、HMS、日志、直播和远程控制能力均按设备所属 `workspace_id` 隔离。
4. 超级管理员只能在 `/admin` 平台管理体系跨租户管理，不影响普通业务前端权限。
5. 后台线程可以处理设备注册和 MQTT/OSD 消息，但最终落库与推送仍必须带正确租户归属。

## 2. 测试范围

| 模块 | 覆盖内容 |
| --- | --- |
| 租户与工作空间 | 租户创建、启停、独立 `workspace_id`、绑定码 |
| RBAC | 租户管理员、项目经理、班组长、普通成员、超级管理员 |
| 组织 | 项目、班组、用户、角色分配 |
| 设备 | 机场、无人机、负载、绑定、解绑、拓扑、详情 |
| 实时数据 | WebSocket、MQTT、OSD、在线/离线、HMS、事件 |
| 业务数据 | 航线、任务、飞行历史、媒体、检测结果、报告、告警、日志、反馈 |
| 前端缓存 | 登录切换、菜单刷新、工作空间刷新、本地缓存清理 |
| 模拟器 | 虚拟机场绑定到指定租户、自动 OSD、跨租户验证 |

## 4. 通用判定标准

可接受结果：

- 返回 `403`
- 返回 `404`
- 返回空数据
- 返回统一业务错误
- 前端显示无权限、无数据或资源不存在

不可接受结果：

- 返回其他租户真实数据
- 修改其他租户数据成功
- 下拉框、筛选器、地图、缓存或实时消息暴露其他租户名称、用户、项目、班组、设备、媒体或告警
- 普通业务前端继承超级管理员跨租户能力

| MT-BG-003 | 后台线程不能写入空 workspace 的业务数据 | 已绑定设备发布 payload/OSD/HMS | 查询相关表 | 业务数据带设备 workspace；普通租户隔离生效 | DB、接口响应 |
| MT-BG-004 | 后端重启恢复绑定设备订阅 | 已绑定设备处于模拟器自动 OSD | 重启 DISys 后端 | 已绑定网关重新注册 SDK 订阅，后续 OSD 正常处理 | Docker 日志、OSDPanel |

## 16. 自动化建议

建议优先将以下用例自动化：

1. `MT-DEV-004` 到 `MT-DEV-006`：设备列表和详情隔离。
2. `MT-RT-001` 到 `MT-RT-003`：OSD WebSocket 隔离。
3. `MT-ORG-001` 到 `MT-ORG-006`：项目、班组、用户列表隔离。
4. `MT-MEDIA-001` 到 `MT-MEDIA-007`：媒体、检测结果、报告、告警隔离。
5. `MT-AUTH-005`、`MT-FE-002`：账号切换和缓存清理。

自动化脚本应记录：

- 登录账号、角色、workspace
- 请求 URL、方法、关键请求体
- HTTP 状态码、业务 `code`、响应摘要
- WebSocket 连接 workspace、收到的 `biz_code` 和设备 SN
- 数据库关键字段：`workspace_id`、设备 SN、任务 ID、媒体 ID

## 17. 测试退出标准

满足以下条件后可认为多租户基础能力通过验收：

1. MT-01 到 MT-14 核心验收项全部通过。
2. 三租户设备列表、设备详情、OSD WebSocket、项目、班组、用户、航线、任务、媒体、告警均无跨租户真实数据返回。
3. 超级管理员跨租户能力只出现在 `/admin` 体系。
4. 浏览器切换账号后无旧租户菜单、设备、项目、OSD 残留。
5. 后端重启后已绑定模拟器机场仍能恢复 OSD 链路。
6. 所有失败项均有缺陷单或风险说明，且不包含 P0/P1 隔离缺陷。
