# 跨服务链路

## 1. 链路目标

本文件用于统一描述平台内主要服务之间的业务调用方向，避免同一条链路在三个仓库中各写一部分而互相不一致。

## 2. 主要调用方向

```text
前端 ─HTTP──▶ DISys
前端 ─HTTP──▶ Detection Server
前端 ─WebSocket──▶ Detection Server
前端 ─STOMP WS──▶ DISys
前端 ─MQTT──▶ EMQX

Detection Server ─HTTP──▶ DISys
Detection Server ─S3──▶ MinIO
Detection Server ─HTTP──▶ SRS

DJI 设备 ─MQTT──▶ EMQX ─▶ DISys
DJI 设备 ─RTMP──▶ SRS ─▶ 前端 / Detection Server
```

## 3. 关键链路说明

### 前端 -> DISys

- 负责主业务数据获取与提交
- 包含用户认证、设备、任务、航线、媒体、告警等主业务接口
- 开发环境通常通过 Vite 代理前缀访问

### 前端 -> Detection Server

- 负责检测会话启动、停止、状态查询、历史结果浏览
- 主要面向检测服务特有能力，不承载平台主业务持久化

### Detection Server -> DISys

- 负责检测会话注册、结果回调、告警回调、媒体登记、报告任务同步
- 该链路字段变化时，最容易出现跨仓库不一致

### Detection Server -> MinIO

- 负责保存截图、录像、报告和衍生媒体产物
- 业务层通常只保存对象键或业务引用，不建议过早固化临时访问 URL

### 前端 -> EMQX / DISys WebSocket / Detection WebSocket

- 三类实时链路分别承担设备遥测、控制反馈和检测结果推送
- 联调问题常见于认证、重连、消息格式和页面状态恢复

## 4. 联调时必须同时检查的点

### 检测链路

- 前端 API 封装是否仍按当前请求字段发起请求
- Detection Server 回调字段是否仍与 DISys DTO 保持兼容
- 前端展示层是否兼容驼峰和下划线字段差异
- 会话状态变化是否能在前端和后端同时闭环

### 告警链路

- Detection Server 的触发字段、级别、截图键名
- DISys 的持久化接口和返回结构
- 前端告警列表与详情展示字段

## 5. 版本记录要求

凡是以下链路发生行为变化，都应同步更新 `docs/versions/联调版本清单.md`：

- 检测启动/停止协议
- 报告生成与回调协议
- 告警字段与状态流转
- 媒体登记方式
- 鉴权方式或路由前缀
